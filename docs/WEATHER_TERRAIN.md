# Grimharvest · 天气与地形系统设计

> 目标：让"田野"成为真正的战场变量。地形决定**你站在哪**有意义，天气决定**这一刻整局**的基调。两者都接入既有飞轮（魂收割 / 恐惧潮汐 / 月之母题，见 [DESIGN.md](DESIGN.md)），并 100% 数据驱动（数值进 `src/data/`，逻辑不碰数值）。

本文档可直接作为开发目标（`/goal`）。实现分两个里程碑：**M8 地形** → **M9 天气**。

---

## 一、地形系统 Terrain

### 1.1 基本架构：区块化无限田野

- 世界是**无限**的，按 **区块 chunk**（如 16×16 格，每格 24 世界单位）按需生成。
- 每个 chunk 的地形由**全局种子 + chunk 坐标**经自实现 PRNG 确定性生成（接入每日种子挑战，同种子同地图）。
- 只生成/保留玩家附近的 chunk（如 5×5），离开即回收 → 走对象池，零分配热路径。
- 渲染：每个 chunk 的静态地表**预渲染到离屏 canvas**，主循环只 `drawImage` 区块（架构红线 5），动态效果（被收割、泥泞）单独叠加层。

### 1.2 地形类型表（落点 `src/data/terrain.ts`）

| 地形 | 视觉 | 对玩家 | 对敌人 | 主题钩子 |
|------|------|--------|--------|----------|
| **平野 Plain** | 暗绿草地 | 正常 | 正常 | 基础地表 |
| **麦浪 Crop（可收割）** | 高金黄麦子 | 略减速、视野被遮 | 减速、**暂时丢失仇恨** | 走过/武器扫过会**被割倒**，露出地面并**爆少量净魂**——"收割土地" |
| **泥沼 Bog** | 深褐泥地 | 减速 35% | 减速 50% | 雨天会扩张；困住冲撞者 |
| **枯骨地 Boneyard** | 灰白碎骨 | 正常 | 在此**刷怪概率↑**、未收割魂**复生更快** | 危险刷怪源，逼你别恋战 |
| **腐土 Blight** | 紫黑龟裂 | **持续小掉血** | 敌人在此**移速/攻击↑** | 血月天气下扩张；高风险高密度 |
| **残垣 Ruins** | 石墙/碎柱 | **阻挡移动** | 阻挡移动与投射物 | 战术掩体：卡住敌人、挡吐息者弹道 |
| **圣地 Hallowed** | 苍白石环+微光 | **缓慢回血**、魂不会复生 | 敌人**减速**、不在此刷新 | 稀有安全屋，喘息点 |
| **月池 Moonwell** | 倒映月光的水洼 | 不可通行（绕行） | 不可通行 | 投射物飞越；月相满时**月镰范围↑** |

设计意图：每种地形给"**站位**"一个理由——贪麦浪的收割奖励 vs 怕枯骨刷怪；用残垣挡吐息者 vs 被泥沼困住；抢圣地回血 vs 离刷怪点太远。

### 1.3 与现有系统的集成点

- **移动**：`Player.update` / `Enemy.update` 末尾乘以"脚下地形的 speedMul"。需要 `world.terrainAt(x,y)` 查询（O(1) 按格取）。
- **持续效果**：腐土掉血 / 圣地回血 → 在 `world.update` 对玩家所在格结算（接入既有 `drainPerSec` 思路）。
- **刷怪偏置**：`spawnWave` 选点时按地形权重（枯骨地刷得多、圣地不刷）。
- **魂复生**：`updateSouls` 复生判定读地形（枯骨地窗口×0.5、圣地不复生）。
- **可收割麦浪**：武器命中 / 玩家经过时把该格 Crop 标记为"已割"，落少量净魂，改变渲染层。
- **碰撞**：残垣/月池 = 阻挡格，移动时做格子级碰撞推回（玩家与敌人都受限）。

## 二、天气系统 Weather

### 2.1 基本架构

- 全局**天气状态机**：当前天气 + 剩余时长 + 到下一种天气的过渡（淡入淡出，避免突变）。
- **天气序列**由**种子 + 时间 + 恐惧值**共同驱动：
  - 基础按时间推进的天气表（开局晴朗 → 中段起雾/落雨 → 后段风暴）。
  - **恐惧潮汐**高时**有概率触发血月**（与支柱三耦合：你打得越疯，天越红）。
- 天气是**全局乘区/事件**，叠加在地形之上。

### 2.2 月相 Moon Phase（贯穿母题）

- 一局内月亮从**新月→满月**缓慢推进（或按天气切换）。
- **满月**时：月镰范围/伤害↑、使魔蝙蝠数值↑、月池效果增强、魂吸附半径↑。
- **新月（最暗）**时：视野最小、敌人潜行（仇恨范围↓你这边吃亏），但魂经验×。
- HUD 顶部用一颗会盈亏的小月亮显示当前月相（呼应武器线）。

### 2.3 天气类型表（落点 `src/data/weather.ts`）

| 天气 | 视觉 | 玩法效果 | 触发 |
|------|------|----------|------|
| **晴夜 Clear** | 群星 + 现有萤尘 | 基线 | 开局默认 |
| **浓雾 Fog** | 暗角收紧、远处变灰 | **视野↓**：敌人贴近才看清（埋伏感）；吐息者更难发现 | 中段、低洼地图 |
| **落雨 Rain** | 斜雨线 + 地面溅点 | 泥沼扩张、**全体移速略↓**；偶发**闪电**短暂全屏照亮 | 中段 |
| **雷暴 Storm** | 暴雨 + 频繁雷击 | **落雷**砸向随机点/精英，造成范围伤害（敌我皆中→走位博弈） | 雨后升级 |
| **血月 Blood Moon** | 全屏血红浸染、月变红 | 敌人**更快更猛 + 刷怪↑**，但**掉魂↑、稀有掉落↑**；腐土扩张 | **高恐惧**概率触发（支柱三事件） |
| **霜风 Frostwind** | 冷蓝调 + 飘雪 | 全体**减速**、月池结冰（变可通行冰面，但滑） | 后段/特定种子 |
| **收割之风 Harvest Gale** | 金色风纹横扫 | 风把**魂与投射物**朝一个方向吹（动态吸附偏移）；麦浪整片倒伏 | 稀有正面事件 |

设计意图：天气把"整局基调"周期性洗牌——血月是高风险高收益的狂欢，雾是紧张的潜行，雷暴是额外的环境博弈，收割之风是天降福利。每种都改变你**怎么打这一段**。

### 2.4 天气 × 地形 互动矩阵（涌现）

- 落雨/雷暴 → **泥沼扩张**（地形随天气动态变化）。
- 霜风 → **月池结冰**变成可走但打滑的冰面。
- 血月 → **腐土蔓延**，枯骨地复生更快。
- 满月 + 月池 → 月镰/使魔强化叠加（build 与环境共振）。
- 收割之风 → 全图**麦浪倒伏**，瞬间清出大量净魂（配合高吸附半径暴富）。

## 三、与核心飞轮的咬合

```
恐惧潮汐(高) ──触发──> 血月 ──> 敌人更猛 + 掉魂更多
     ▲                                  │
     │                                  ▼
   杀得更疯 <── 更强 build <── 更多魂收割 <── 抢收(站位×地形×天气)
```

- 地形给**空间**维度的"贪还是稳"，天气给**时间**维度的"贪还是稳"，与既有的魂收割飞轮三方咬合。
- 全部吃同一**每日种子**：同一天所有人面对同样的地图 + 天气序列 → 公平打榜。

## 四、数据落点（`src/data/`）

| 文件 | 内容 |
|------|------|
| `terrain.ts` | 地形类型表：speedMul、掉血/回血、刷怪权重、复生倍率、是否阻挡、是否可收割、渲染色 |
| `terrain-gen.ts`（或 data 表 + core 逻辑） | 区块生成参数：各地形噪声阈值/簇大小/分布权重 |
| `weather.ts` | 天气类型表：视觉参数、各类乘区/事件、持续时长区间 |
| `weather-schedule.ts` | 天气序列表（时间×恐惧 → 天气）、月相推进速度 |

逻辑落点：`core/terrain-chunk.ts`（生成+预渲染区块）、`systems/terrain.ts`（查询+动态变化+碰撞）、`systems/weather.ts`（状态机+全局效果）。

## 五、实现里程碑（给 /goal）

### M8 · 地形系统
1. 区块化无限田野：seed 生成 + 离屏预渲染 + 视野裁剪 + 回收池
2. `terrain.ts` 数据表 + `world.terrainAt(x,y)` 查询
3. 移动减速 / 持续掉血回血 / 刷怪偏置 / 复生倍率接入
4. 可收割麦浪（割倒 → 掉魂 → 改渲染）
5. 阻挡格碰撞（残垣 / 月池）
6. 验收：跑一局确认地形影响走位、性能不掉（2000 敌人 @60fps 仍达标）

### M9 · 天气系统
1. 天气状态机 + 过渡淡入淡出 + `weather.ts` 数据表
2. 月相推进 + HUD 月亮 + 月相对月镰/使魔/吸附的乘区
3. 各天气视觉层（雨/雪/雾/血月浸染/落雷）走粒子池与覆盖层
4. 天气 × 地形动态互动（泥沼扩张、月池结冰、腐土蔓延）
5. 血月接入恐惧潮汐事件；收割之风的魂/投射物风偏
6. 验收：恐惧打满能触发血月；同种子天气序列可复现

## 六、性能红线（务必遵守）

- 区块静态层**预渲染离屏**，主循环只 `drawImage`；动态变化（割麦/泥泞/结冰）改"覆盖标记层"，不重绘整块。
- `terrainAt` 必须 **O(1)**（按世界坐标取格，不做遍历）。
- 天气粒子（雨/雪/雷）全部走**对象池**，复用既有 `Particle` 思路，热路径零分配。
- chunk 生成放在进入新区块时（偶发），不在每帧。
